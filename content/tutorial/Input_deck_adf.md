The ADF is a generalisation of EPOCH's classic distribution functions
but using [calculable particle
properties][Calculable_particle_properties] to specify
the axes of the distribution function and the value accumulated on the
grid for that distribution function.

# Example deck block {#example_deck_block}

```perl
begin:adf
   name= x_px_df
   height = dir_n
   accumulate_mode = sum
   direction=(dir_x, dir_px)
   resolution = (null, 100)
   range = ((0,1), (0,2e-22))
   use_weight = T
   use_shapes = T
end:adf
```

# Name

The ADF must be named so that it can be referred to in pipeline blocks

# Height

The "height" key is a calculable particle property that should be
accumulated on the grid

# Accumulate mode {#accumulate_mode}

The "accumulate_mode" key is how the height should be accumulated on
the grid. Options are

-   sum - Sum the value on the grid
-   average - Calculate the average value of the height in each cell
-   temperature - Ignore the height value and calculate the temperature
    in each cell

If this key is absent then sum is assumed

# Direction

The "direction" key specifies the calculable particle property for
each axis of the distribution function. You may specify up to 6 axes,
specifications for resolutions and ranges must be of the same length as
the direction specification

# Resolution

The "resolution" key specifies the desired resolution for each axis of
the distribution function. The number of keys must match the number of
directions specified. The resolution is ignored for spatial dimensions
(but not resampled spatial dimensions) which must match the base grid
resolution

# Range

The "range" key is a list of (lower_bound, upper_bound) pairs for
each axis of the ADF. There must be as many pairs as there are
directions. If this key is missing then the range is calculated at each
output to contain all particles in the source for the ADF

# Use weight {#use_weight}

The "use_weight" keyword is used to specify whether or not the
particle weight should be used when accumulating the data for the ADF.
For "height=dir_n" this key being F means that you are calculating
the number of simulation particles in each cell of the ADF. It's
meaning is less clear for other accumulated heights. If this key is not
specified it is assumed to be T

# Use shape function {#use_shape_function}

The "use_shape" keyword specified whether or not to use the shape
function when calculating spatial axes for distribution functions. If F
then all of a particle's property is deposited in the cell which
contain the particle's centre. This is comparable to the method used by
the EPOCH legacy distribution functions.
